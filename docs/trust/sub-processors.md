# Ghostlight Subprocessors

Ghostlight engages no subprocessors. This page is short because there is nothing to list.

No third party receives customer data on Ghostlight's behalf, because the vendor itself
receives no customer data. There is no vendor-side service to pass data along to, so the
category of subprocessor does not arise. The only third parties anywhere in the picture are
ones you select and control: the MCP client and model you run, the SIEM or log destination
you stream audit to, and the endpoint that hosts your central policy. Those are your
relationships and your choices, not Ghostlight's subprocessors.

If this ever changes, the change will be announced in release notes and will be visible in
this file's git history. Until then, the correct answer to a subprocessor question is: none.
